Artículo que resuelve la estructura de la tabla de artículos en MySQL
donde supuestamente no pueden haber dos tablas con timestamp por defecto:

https://stackoverflow.com/questions/4489548/why-there-can-be-only-one-timestamp-column-with-current-timestamp-in-default-cla

Las solución fue esta:

CREATE TABLE `artículos` (
  `artículoId` int(11) NOT NULL,
  `firstTS` timestamp NOT NULL DEFAULT '0000-00-00 00:00:00',
  `lastTS` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `título` varchar(255) COLLATE utf8_spanish_ci NOT NULL,
  `cuerpo` text COLLATE utf8_spanish_ci NOT NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COLLATE=utf8_spanish_ci;

Y al meter los datos usar NULL en los campos que se deben actualizar a las TimeStamp actuales
